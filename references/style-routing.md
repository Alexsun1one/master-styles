# Subject → Register Routing

The register is chosen BY THE SUBJECT, automatically. The user never names a master. Read the subject, match the row, pull that row's tokens. When two rows fit, pick the one a viewer of this subject EXPECTS (a Song-dynasty poet belongs on rice paper, never a neon grid).

Read this file FIRST — before `references/master-library.md`. It tells you which master entries to open; the library is the token detail for the row you land on.

Rule: route to ONE register, apply its tokens, keep one restraint. If invoked as an engine under a host skill, the host's restraint overrides any maximalist row (see the restraint-compatibility gate in SKILL.md). The user may override; default is the routed register, never a generic gradient.

## How to route

1. Read the subject + domain + mood.
2. Match the row. When two fit, pick the register the subject's own audience expects.
3. Pull that row's token sources, then open those masters in `references/master-library.md` for the detail.
4. Keep the row's one restraint through the whole image.
5. If running under a host (cover, quote-card, rank-card), check the host's anti-pattern list first; a maximalist row is INCOMPATIBLE with an anti-clutter host — pick a different master from the same feeling row.
